# 零出力场站排查报告

**日期**：2026-06-25  
**排查范围**：`RQ0`、`RQ1`、`ref_code/calculate_bcsd_cfs`、`ref_code/calculate_wind_solar_out`

---

## 一、问题描述

在 `RQ1/plot_RQ1_cf.py` 和 `RQ1/plot_RQ1_cf_capacity_quadrant.py` 中，绘图前均过滤了 `CF = 0` 或 `generation = 0` 的场站。2050 年被过滤的零出力场站占比如下：

| 技术 | 情景 | 零出力场站占比 | 零出力装机占比 |
|------|------|----------:|----------:|
| 光伏 | SSP1-2.6 | 21.7% | 21.5% |
| 光伏 | SSP2-4.5 | 23.6% | 19.6% |
| 光伏 | SSP5-8.5 | 31.6% | 44.1% |
| 风电 | SSP1-2.6 | 25.1% | 22.1% |
| 风电 | SSP2-4.5 | 26.8% | 24.4% |
| 风电 | SSP5-8.5 | 24.4% | 20.5% |

2030、2040 年的比例相近（分别约 25.2%、25.5%），说明该问题并非随年份而改变。

---

## 二、排查结论

**根本原因：BCSD 气候数据在部分陆地格点上存在 NaN 缺失值，CF 计算脚本将 NaN 辐照度/风速转换为 0，导致这些格点的 CF 输出为 0 而非 NaN，最终使落在这些格点上的场站年出力为零。**

---

## 三、根因溯源：完整传播链

### 步骤 1：BCSD 源数据存在系统性 NaN

BCSD 降尺度输出（`rsds_bcsd`、`uas_bcsd`、`vas_bcsd`）在 0.1° 陆地格点上存在大量全时段 NaN。以澳大利亚为例：

- BCSD 格点总数：140,415 个
- 全时段 NaN 格点：87,277 个（**占比 62.2%**）
- 有效数据格点：53,138 个（37.8%）

**原因**：BCSD 输出网格（0.1°，按陆地裁剪）与底层 NESM3 气候模型的陆海掩膜不一致。NESM3 在其原始分辨率下将部分沿海格点视为"海洋"，因此没有对应数据；BCSD 降尺度后这些格点仍为 NaN，无法生成有意义的 CF 值。

NaN 格点在澳大利亚的地理分布不均匀，主要集中在南部（低于 39°S 段 NaN 占比 87–100%）和北部（高于 14°S 段 NaN 占比 73–100%），即气候模式陆地覆盖较差的区域。

### 步骤 2：CF 计算脚本将 NaN 静默转为 0

`S01E01_Simulate_Solar_CF_BCSD.py` 中：
```python
def to_kw_m2(arr, units):
    ...
    out = np.nan_to_num(out, nan=0.0, posinf=0.0, neginf=0.0)  # NaN → 0
    return np.maximum(out, 0.0)
```

`S01E02_Simulate_Wind_CF_BCSD.py` 中：
```python
def wind_to_ms(arr, units):
    ...
    x = np.nan_to_num(x, nan=0.0, posinf=0.0, neginf=0.0)  # NaN → 0
    return x
```

NaN 辐照度被转为 0 W/m²，NaN 风速被转为 0 m/s，均导致 CF = 0。

### 步骤 3：陆地掩膜未能拦截这些格点

CF 计算最后一步对海洋格点赋 NaN：
```python
cf_chunk = np.where(land_mask[None, :, :], cf_chunk, np.nan)
```

但由于 BCSD 输出网格**本身已按陆地裁剪**，`global_land_mask` 将网格内所有格点都识别为陆地（True），不会将任何格点置为 NaN。因此 CF = 0 的格点被完整保留。

**验证**：澳大利亚 CF 文件中，87,277 个零值格点与 BCSD NaN 格点**完全重合（100% 重叠）**，无例外。巴西、西班牙、印度的验证结果一致（重合度均为 100%）。

### 步骤 4：场站匹配到零值格点

`station_output_calculator_0p1deg.py` 对场站做最近邻配准（`match_dist ≈ 0°`），场站坐标落在 CF = 0 的格点上，全年每个时步的发电量均为 0（不是 NaN）。

检查匹配距离确认这不是配准失败：零出力场站的 `match_dist_deg` 中位数 ≈ 0.000003°，远低于容差 0.15°。

### 步骤 5：年度 CF 统计得零

`prepare_RQ1_data.py` 在计算年 CF 时：
```python
pwr_sum = np.nansum(yr_power[:, idx], axis=0)  # 对全年零值求和 = 0
ann_cf  = pwr_sum / (n_yr_steps * cap_gw[idx]) # 0 / anything = 0
```

`has_data` 过滤器只排除**全为 NaN** 的场站；全为 0 的场站仍被保留，输出 CF = 0、年发电量 = 0。

---

## 四、已排除的假设

| 假设 | 排查结果 |
|------|---------|
| 场站与 CF 格点配准失败 | 否。零出力场站 `match_dist ≈ 0°`，配准正常 |
| activation_year > 目标年 | 否。零出力场站中 activation_year 2030、2040 均有，到 2050 早已激活 |
| 数据被填成零（fill_value=0） | 否。BCSD 原始数据以 NaN 表示缺失，CF 脚本误转为 0 |
| 场站落在无效 CF 网格外 | 否。场站坐标落在 CF 文件覆盖范围内，只是数据为 0 |
| 某些区域 CF 数据文件缺失 | 否。文件存在，读取正常，仅内部值为 0 |

## 六、结论与建议

### 结论

零出力场站并非物理上真实的"全年不发电"现象，而是 **BCSD 气候数据在 NESM3 模型陆海掩膜与 0.1° 输出网格不一致处产生的系统性数据缺口**，经 `nan_to_num(..., nan=0.0)` 静默传播为零 CF，最终在场站层面体现为零出力。

### 处理建议

当前绘图脚本（`plot_RQ1_cf.py`、`plot_RQ1_cf_capacity_quadrant.py`）已正确地将这些场站过滤出可视化，处理思路合理。若需进一步改进数据质量，可在 CF 计算步骤将 BCSD NaN 格点改为输出 NaN（而非 0），使其在后续统计中被 `has_data` 过滤掉，从而避免噪声场站进入 `station_annual_generation.csv`。具体改动为：

在 `S01E01_Simulate_Solar_CF_BCSD.py` 和 `S01E02_Simulate_Wind_CF_BCSD.py` 中，在 CF 计算前先建立 BCSD 有效值掩膜，计算完 CF 后将原始 NaN 格点恢复为 NaN（而非沿用 `nan_to_num` 的 0 值）：

```python
# 记录哪些格点在源数据中是NaN（气候模式无数据）
bcsd_valid = ~np.isnan(rsds_raw)  # 或 uas_raw

# 计算CF...
cf_chunk = compute_solar_cf_chunk(...)

# 恢复BCSD缺失格点为NaN（而非让nan_to_num的0传播）
cf_chunk = np.where(bcsd_valid, cf_chunk, np.nan)

# 陆地掩膜（去除海洋格点）
cf_chunk = np.where(land_mask[None, :, :], cf_chunk, np.nan)
```

这样 `station_output_calculator_0p1deg.py` 中的 `np.ma.filled(cf_slab, np.nan)` 会正确保留 NaN，场站的 `has_data` 过滤器便可在 `prepare_RQ1_data.py` 中将这些场站完全排除。
